# AI Skill 验证与测试指南

## 🧪 验证 Skill 执行效果的方法

### 1. 本地测试验证
**直接调用测试**
```bash
# 测试技能是否能正确响应
molt ask "使用 slidev 技能创建一个演示文稿"

# 验证工具调用
molt ask "帮我查找关于大模型的最新技术文章"
```

**参数验证**
- 检查输入参数是否被正确解析
- 验证边界条件和错误处理
- 确保输出格式符合预期

### 2. 功能完整性测试
**端到端流程验证**
1. **触发条件**: 确认 Skill 在正确场景下被激活
2. **执行过程**: 验证所有步骤按预期执行
3. **结果输出**: 检查最终结果是否满足需求
4. **错误处理**: 测试异常情况下的表现

**示例测试场景**
```yaml
# 测试 Slidev Skill
输入: "帮我创建一个关于 React 18 新特性的演示文稿"
期望:
  - 创建 knowledge/react-18-features 目录
  - 生成 slides.md 文件
  - 包含正确的 Frontmatter 配置
  - 可以正常启动开发服务器
```

### 3. 性能和用户体验验证
**响应时间**
- 工具调用是否在合理时间内完成
- 复杂任务是否有进度反馈
- 大文件处理是否有流式输出

**交互体验**
- 提示信息是否清晰友好
- 错误信息是否具有指导性
- 是否提供有用的后续建议

### 4. 集成测试
**与其他 Skill 的协作**
- 验证多个 Skill 能否协同工作
- 检查上下文传递是否正确
- 确保资源不会冲突

**与现有工作流集成**
- 验证是否符合团队的开发规范
- 检查是否与现有工具链兼容
- 确保输出格式便于后续处理

## 🔍 具体验证步骤

### 步骤 1: 基础功能验证
```bash
# 1. 检查技能是否被正确加载
molt skills list | grep your-skill-name

# 2. 测试基本调用
molt ask "使用 [你的技能名] 做某事"

# 3. 验证输出结果
检查生成的文件、执行的命令、返回的信息
```

### 步骤 2: 边界条件测试
- **空输入**: 测试没有提供必要参数时的行为
- **无效输入**: 测试错误或不支持的参数
- **超长输入**: 测试大量数据的处理能力
- **并发调用**: 测试同时调用多个实例的情况

### 步骤 3: 实际场景测试
**模拟真实使用场景**
- 在实际项目中使用你的 Skill
- 让团队成员试用并收集反馈
- 在不同环境下测试（不同操作系统、网络条件等）

### 步骤 4: 持续监控
**部署后的监控**
- 记录 Skill 的使用频率和成功率
- 收集用户反馈和改进建议
- 监控性能指标和错误率

## 📊 验证指标

### 成功率指标
- **调用成功率**: Skill 被成功调用的比例
- **任务完成率**: 完整完成预期任务的比例  
- **用户满意度**: 用户对结果的满意程度

### 性能指标
- **响应时间**: 从调用到返回结果的时间
- **资源消耗**: CPU、内存、网络等资源使用情况
- **可扩展性**: 处理大规模任务的能力

### 质量指标
- **准确性**: 输出结果的正确性
- **一致性**: 相同输入产生相同输出
- **可靠性**: 在各种条件下都能稳定工作

## 🛠️ 调试技巧

### 日志分析
```bash
# 查看详细执行日志
molt --verbose ask "使用你的技能"

# 检查工具调用详情
molt tools list --verbose
```

### 逐步调试
1. **隔离测试**: 单独测试 Skill 的每个组件
2. **模拟环境**: 在受控环境中测试特定场景  
3. **回滚机制**: 准备快速回滚方案以防出现问题

### 常见问题排查
- **权限问题**: 检查文件系统和网络权限
- **依赖问题**: 确保所有依赖都已正确安装
- **配置问题**: 验证配置文件的正确性
- **版本兼容**: 检查工具版本的兼容性
