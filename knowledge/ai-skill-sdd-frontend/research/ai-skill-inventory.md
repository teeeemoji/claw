# AI Skill 清单：按需求类型分类

## 🎯 需求类型 1：文案微调类（占比 ~30%）

### 文案替换 Skill
- **功能**: 自动识别和替换指定文案
- **输入**: 原文案、新文案、上下文范围
- **输出**: 修改后的代码文件
- **示例**: "将'立即购买'改为'马上抢购'"

### 国际化文案 Skill  
- **功能**: 自动处理多语言文案替换
- **输入**: 语言包、文案映射表
- **输出**: 更新后的 i18n 文件和组件
- **示例**: 同时更新中英文文案

### A/B 测试文案 Skill
- **功能**: 自动创建 A/B 测试文案变体
- **输入**: 原文案、测试变体、实验配置
- **输出**: 带实验逻辑的代码
- **示例**: 创建多个按钮文案进行 A/B 测试

## 🎯 需求类型 2：开关控制类（占比 ~25%）

### 功能开关 Skill
- **功能**: 自动添加/修改功能开关逻辑
- **输入**: 开关名称、默认状态、条件逻辑
- **输出**: 带开关控制的代码
- **示例**: "开启新用户注册流程"

### 权限控制 Skill
- **功能**: 自动添加权限检查逻辑
- **输入**: 权限点、用户角色、访问规则
- **输出**: 带权限控制的组件
- **示例**: "仅管理员可见的管理按钮"

### 环境配置 Skill
- **功能**: 自动处理环境相关的开关
- **输入**: 环境变量、配置项、默认值
- **输出**: 环境感知的代码
- **示例**: "开发环境显示调试信息"

## 🎯 需求类型 3：样式调整类（占比 ~20%）

### 样式微调 Skill
- **功能**: 自动修改 CSS/SCSS 样式
- **输入**: 选择器、样式属性、新值
- **输出**: 更新后的样式文件
- **示例**: "将按钮颜色从蓝色改为红色"

### 响应式适配 Skill
- **功能**: 自动添加响应式样式
- **输入**: 断点、设备类型、样式规则
- **输出**: 响应式 CSS 代码
- **示例**: "移动端隐藏某个组件"

### 主题切换 Skill
- **功能**: 自动处理多主题样式
- **输入**: 主题变量、样式映射
- **输出**: 主题化的组件代码
- **示例**: "支持深色/浅色主题切换"

## 🎯 需求类型 4：逻辑增强类（占比 ~15%）

### 业务逻辑 Skill
- **功能**: 自动添加业务验证和处理逻辑
- **输入**: 业务规则、验证条件、处理流程
- **输出**: 增强后的业务逻辑代码
- **示例**: "添加手机号格式验证"

### 数据处理 Skill
- **功能**: 自动添加数据转换和处理逻辑
- **输入**: 数据结构、转换规则、错误处理
- **输出**: 数据处理函数和组件
- **示例**: "格式化日期显示为 'YYYY-MM-DD'"

### 事件处理 Skill
- **功能**: 自动添加事件监听和处理
- **输入**: 事件类型、处理逻辑、防抖配置
- **输出**: 事件处理代码
- **示例**: "点击按钮后发送埋点"

## 🎯 需求类型 5：完整功能类（占比 ~10%）

### 组件生成 Skill
- **功能**: 从零生成完整 UI 组件
- **输入**: 设计稿、交互说明、数据模型
- **输出**: 完整的 React/Vue 组件
- **示例**: "创建商品详情页组件"

### 页面搭建 Skill
- **功能**: 自动生成完整页面
- **输入**: 页面结构、路由配置、数据源
- **输出**: 完整的页面代码
- **示例**: "创建用户个人中心页面"

### API 集成 Skill
- **功能**: 自动生成 API 调用和数据处理
- **输入**: API 文档、请求参数、响应处理
- **输出**: API 服务和组件集成
- **示例**: "集成用户信息查询接口"

## 🔧 通用辅助 Skill

### 代码定位 Skill
- **功能**: 快速定位需要修改的代码位置
- **输入**: 需求描述、项目结构
- **输出**: 目标文件和行号
- **适用**: 所有需求类型

### 影响分析 Skill
- **功能**: 分析代码修改的影响范围
- **输入**: 修改点、依赖关系
- **输出**: 影响范围报告和测试建议
- **适用**: 所有需求类型

### 测试生成 Skill
- **功能**: 自动生成针对性的测试用例
- **输入**: 修改内容、业务逻辑
- **输出**: 单元测试和 E2E 测试
- **适用**: 所有需求类型

### 回滚保护 Skill
- **功能**: 自动生成回滚方案和备份
- **输入**: 修改内容、风险评估
- **输出**: 回滚脚本和备份文件
- **适用**: 所有需求类型

## 📊 技能组合策略

### 简单需求（文案/开关）
- **技能组合**: 代码定位 + 文案替换/开关控制 + 测试生成
- **执行时间**: < 5 分钟
- **人工介入**: 仅需确认和部署

### 中等需求（样式/逻辑）
- **技能组合**: 代码定位 + 样式调整/逻辑增强 + 影响分析 + 测试生成  
- **执行时间**: 5-30 分钟
- **人工介入**: 需要代码审查和测试验证

### 复杂需求（完整功能）
- **技能组合**: 需求分析 + 组件生成 + API 集成 + 影响分析 + 测试生成 + 回滚保护
- **执行时间**: 30 分钟 - 2 小时
- **人工介入**: 需要完整的设计评审和测试验证

## 🚀 实施建议

1. **优先实现高频技能**: 先实现文案替换和开关控制类 Skill
2. **渐进式推广**: 从简单需求开始，逐步扩展到复杂需求  
3. **质量保证**: 每个 Skill 都需要完整的测试覆盖
4. **团队培训**: 让开发者了解如何正确使用这些 Skill
5. **持续优化**: 基于实际使用反馈不断改进 Skill
