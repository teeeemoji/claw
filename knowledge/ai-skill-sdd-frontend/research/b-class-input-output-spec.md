# B类需求（轻量级UI需求）输入输出规范

## 1. 需求智能分类 (Demand Classification)
**输入**:
- 原始需求描述（"调整按钮颜色为品牌主色"、"增加卡片间距"等）
- 设计系统文档或样式指南
- 项目技术栈信息（CSS-in-JS、Tailwind、SCSS等）

**输出**:
- 需求类型标签（B类-样式微调/B类-布局优化/B类-交互增强）
- 影响范围评估（全局/局部样式）
- 所需设计系统 token 列表

## 2. 样式分析 (Style Analysis)
**输入**:
- 分类后的需求
- 项目样式架构（CSS Modules、Styled Components、Tailwind等）
- 设计系统 token 配置
- 现有组件库结构

**输出**:
- 目标样式文件路径列表
- 相关 CSS 类名或 styled component 列表
- 依赖的 design token 映射
- 潜在的样式冲突预警

## 3. 组件定位 (Component Localization)
**输入**:
- 样式分析结果
- 组件树结构
- 页面路由映射
- 用户行为数据（可选）

**输出**:
- 精确的组件文件路径
- 相关的样式作用域
- 可能受影响的其他组件列表
- 测试用例覆盖建议

## 4. 智能修改 (Intelligent Modification)
**输入**:
- 定位到的组件和样式
- 新的样式规范或设计要求
- 项目样式约束（响应式断点、主题变量等）

**输出**:
- 修改后的组件代码
- 更新的样式文件
- 自动生成的样式文档更新
- 格式化后的代码（遵循项目规范）

## 5. 视觉回归 (Visual Regression)
**输入**:
- 修改后的代码
- 基准截图库
- 视觉测试配置
- 浏览器兼容性要求

**输出**:
- 视觉差异报告
- 回归测试通过/失败状态
- 不同设备/浏览器的预览链接
- 性能影响评估（CSS bundle size等）

## 6. 自动提交 (Auto Commit)
**输入**:
- 视觉回归验证通过的代码
- 提交信息模板（包含视觉变更说明）
- 分支策略和 PR 模板

**输出**:
- Git 提交记录（包含视觉变更截图）
- Pull Request（包含 before/after 对比）
- 部署预览链接（Vercel/Netlify等）
- 自动化测试触发
